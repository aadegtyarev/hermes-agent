# Почему grounding работает (tool-mediated vs declarative)

## Вопрос

«Что мешает модельке врать про заземление? Почему grounding вообще работает, если я могу просто написать «я проверил, всё ок» и сфабриковать команду и её вывод?»

## Ответ

Grounding работает не потому что модель «обещает не врать», а потому что он **инструментальный**, а не декларативный.

### Схема: гарантия через инструменты

```
утверждение → вызов инструмента (ssh_run/execute_code/read_file) → реальный stdout/stderr попадает в контекст → вывод обрабатывается → пользователь может повторить
```

**Что предотвращает fabrication:**

1. **Инструмент, а не риторика.** Модель не говорит «запустила kill -9» — она реально запускает `ssh_run(host, "kill -9 $(MainPID)")`. Вывод команды физически приходит в контекст как `stdout`. Сфабриковать вывод нельзя — в контексте лежит реальный результат вызова.

2. **Воспроизводимость.** Любой, кто прочитает отчёт, может запустить ту же команду и получить тот же результат. Fabrication обнаруживается первым же `systemctl is-active`.

3. **Цифры из измеримых артефактов.** «HEAP +22 MB» — не слова, а результат `mosquitto_sub -t '$SYS/broker/heap/current' -C 1` до и после. Цифра берётся из инструмента, не из памяти модели.

4. **Перекрёстная проверка.** Даже если одна линия сфабрикована — вторая (git log configs, A/B тест, код upstream) её опровергнет. Ложь в одной линии не выдерживает трёх независимых проверок.

### Единственное пространство для ошибки

**Интерпретация вывода инструмента.** Модель может неверно прочитать цифры, неправильно связать причину и следствие, или пропустить фактор, которого нет в данных. Но это уже не fabrication, а **ошибка рассуждения** — и она бьётся перекрёстной проверкой (сабагент с deepseek-v4-pro, второй проход, A/B тест).

### Граница

| Тип | Работает? | Пример |
|-----|-----------|--------|
| Tool-mediated grounding | 🟢 да, потому что реальный вызов → вывод в контекст | `ssh_run(host, "kill -9 …")` → вывод `systemctl is-active` |
| Declarative grounding (слова) | 🔴 нет, модель может сфабриковать | «я проверил, всё ок» без реальных tool calls |
| Перекрёстная верификация | 🟢 да, fabrication не выдерживает 3+ линий доказательств | код + A/B + git log → однозначный корень |
| Интерпретация результата | 🟡 рискованно, требует перепроверки | неверная причина при верных цифрах |

### Итог

Grounding работает, потому что за каждым 🟢 стоит **реальный вызов инструмента**, а не риторическая конструкция. Если бы grounding был просто фразой «я заземлил» — цена ему ноль. Он работает, потому что инструмент приносит в контекст данные из внешнего мира, которые модель не контролирует.
